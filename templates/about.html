About the Dataset

Overview

This repository merges two complementary Pokémon data sources into a single, unified dataset (available in CSV and JSON). It's tailored for building an end-to-end supervised ML model that predicts Pokémon battle outcomes. By keeping only Pokémon common to both sources, the merged dataset offers consistent IDs, names, stats, and battle records—laying the groundwork for accurate, reproducible model training and a polished Pokédex UI.

1. Data Sources

1.1 Kaggle "Pokémon Challenge"
  • URL: https://www.kaggle.com/datasets/terminus7/pokemon-challenge
  • Contents:
    • pokemon.csv – Base characteristics for each Pokémon (ID, name, stats, types, generation, legendary flag)
    • combats.csv – Simulated combat records (attacker ID, defender ID, winner ID)
  • Original Scope: ~800 unique Pokémon; battle data generated by a custom algorithm to simplify in-game mechanics
1.2 Anand Baraik's Pokédex JSON
  • URL: https://github.com/anandbaraik/pokedex
  • Contents:
    • pokeData.ts – Detailed entries per Pokémon, including name, type(s), base stats, and sprite URLs.

2. Merge & Reduction

  • Inner Join: We performed an inner join on Pokémon ID and name, retaining only those present in both sources.
  • Resulting Counts:
    • Pokémon: XX entries (reduced from ~800)
    • Combat Records: YY encounters (filtered to battles where both participants are in the merged roster)

4. Notes & Considerations

  • Dropped Entries: Rare or region-exclusive Pokémon appearing in only one source were excluded.
  • Stat Consistency: Verified that base stats match between sources for each common Pokémon.
  • Battle Simplification: As with the original Kaggle data, these battles omit in-game mechanics (abilities, items, weather, etc.).

Provenance & Collection Methodology

Sources
  1. Kaggle Pokémon Challenge

    "Welcome to Weedle's Cave. The Pokémon characteristics come from the actual game but the battle data has been generated by a custom algorithm that doesn't take into account some game characteristics to simplify the dataset."

    • URL: https://www.kaggle.com/datasets/terminus7/pokemon-challenge
  2. Anand Baraik's Pokédex
    • URL: https://github.com/anandbaraik/pokedex
    • Original data in TypeScript; converted to CSV/JSON for this project.
Methodology
  • Data collected from Kaggle and GitHub.
  • TypeScript files were parsed and transformed into CSV/JSON.
  • Datasets were merged via inner joins on Pokémon ID and name.
